    # 10、JVM，JDK，JRE 三者的之间的联系？

    1.jdk包含jre包含jvm

    2.jdk：有编译、运行时环境

    3.jre：是运行时环境，还有Java核心的类库

    4.jvm是虚拟机：负责执行字节码-变成机器可以识别的机器码，是跨平台的关键

    # 11、方法重载和方法重写区别？

    重载

    1.方法的名称必须一致，

    2.方法中的参数类型、个数、顺序不一致

    3.重载的方法和原来的方法没有关系

    4.可以有不同的返回类型

    5.重载在编译的时候就确定了要调用的方法

    6.在本类中重载

    重写

    1.需要用orrivide修饰

    2.方法的名称和参数类型、个数都必须和重写的方法一致

    3.返回类型也得和重写的方法一致

    4.重写需要在运行时动态的判断调用哪个方法

    5.在子类中重写父类的方法

    # 12、接口和抽象类之间的区别？

    interface abstract
    1.定义 interface修饰的类，定义方法，不实现 abstract修饰的方法，可选择是否实现
    2.继承 一个类可以多继承interface 一个类只能继承一个抽象类
    3.字段 常量：public static final 没有限制
    4.构造函数 因为不能实例化，所以没有构造函数 可以有
    5.访问修饰符 只能是public或abstract 没有限制
    6.实现 implement来实现接口中的方法 extends来实现接口中所有的抽象方法

    # 15、讲一下 HashMap 的源码？

    数组+链表 jdk1.7之前

    1.存储，每次进行put时会计算key的hashcode，得到的哈希码会存放在数组中的某一个位置

    2.相同的hashcode是hash冲突，hash冲突时会把需要存入的元素用链表进行连接

    3.hashmap的默认大小是16，扩容因子是0.75，扩容条件hashmap中键值对个数大于16*0.75，也就是12，每次扩容为原来的2倍大小，

    HashMap在进行扩容时，使用的rehash方式非常巧妙，因为每次扩容都是翻倍，与原来计算的 (n-1)&hash的结果相比，只是多了一个bit位，所以节点要么就在原来的位置，要么就被分配到"原位置+旧容量"这个位置。

    jdk1.8：数组+链表+红黑树（当链表长度大于8且数组长度大于64时会转换成红黑树（平衡查询快），为了解决链表查询速度慢，提高查询效率
